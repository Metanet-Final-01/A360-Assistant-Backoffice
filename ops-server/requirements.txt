# langchain_community.chat_models.vertexai를 무조건 임포트하는데, 그 서브모듈이
# langchain-community 0.4+에서 별도 패키지로 분리돼 없어짐). ragas 0.2.15 +
# langchain-community 0.3.19 + langchain<0.4 조합에서 동작 확인함.
#
# 2026-07-13 재검증: 아래 조합(이전엔 langchain-openai==1.3.3, openai==2.44.0,
# langchain-core 미명시)을 새 venv에 `pip install -r requirements.txt` 그대로
# 설치하면 ResolutionImpossible로 실패하는 걸 재현했다 — langchain-community==0.3.19가
# langchain-core<1.0.0을 요구하는데 langchain-openai==1.3.3은 langchain-core>=1.4.7을
# 요구해서 두 조건을 동시에 만족하는 langchain-core가 존재하지 않음(신세대 langchain-
# openai 1.x를 구세대 langchain-community 0.3.x와 잘못 짝지은 것). 기존 로컬 venv가
# "동작"했던 건 이 조합을 pip install -r로 새로 깐 게 아니라, 이후 개별 패키지를
# 하나씩 업그레이드하면서 우연히 충돌 검증을 건너뛴 상태로 굳어진 것 — requirements.txt
# 파일 자체로는 애초에 재현 불가능했다(신규 개발자가 README대로 설치하면 100% 실패).
#
# langchain-openai를 구세대(langchain-core<1.0.0과 호환되는 0.x)로 낮춰서 고정하고,
# langchain-core도 명시 고정해 새 venv에서 실제로 깨끗하게 설치되고 ragas/langchain_openai
# 임포트도 정상 동작하는 걸 재검증했다.
